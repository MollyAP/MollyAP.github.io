Class Notes

An Introduction to Node.js on sitepoint.com

  1. What does REST stand for?
    2. Representational State Transfer

  2. REST APIs are designed around a ____.
    3. Resources

  3. What is an identifier of a resource? Give an example.
    4. A string of characters used to identify or locate a resource. Uniform Resource Locator (URL).

  4. What are the most common HTTP verbs?
    5. GET, POST, PUT, PATCH, and DELETE

  5. What should the URIs be based on?
    6. Nouns, aka the resource

  6. Give an example of a good URI.
    7. https://molly.com/api/users/42069

  7. What does it mean to have a ‘chatty’ web API? Is this a good or a bad thing?
    8. When your web APIs expose a large number of small resources. A bad thing.

  8. What status code does a successful GET request return?
    9. A 200 (OK) status

  9. What status code does an unsuccessful GET request return?
    10. A 404 (Not Found) code

  10. What status code does a successful POST request return?
    11. A 201 (Created) code

  11. What status code does a successful DELETE request return?
    12. 204 (No Content) code
