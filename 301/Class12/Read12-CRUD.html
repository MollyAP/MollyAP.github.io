Status Codes Based On REST Methods

  1. In your own words, describe what each group of status code represents:
    • 100’s =The server has received the request and is continuing the process.
    • 200’s =The server successfully processed the request.
    • 300’s =The client needs to take further action to complete the request.
    • 400’s =There was an error on the client-side.
    • 500’s =There was an error on the server-side.


  2. What is a status code 202?
    3. The request was valid, but its processing will finish sometime in the future.

  3. What is a status code 308?
    4. The resource will now be available at a new URL and the client should directly access it via the new URL in the future.

  4. What code would you use if an update didn’t return data to a client?
    5. 204 No Content.

  5. What code would you use if a resource used to exist but no longer does?
    6. 410 Gone.

  6. What is the ‘Forbidden’ status code?
    7. 403.

Build A REST API With Node.js, Express, & MongoDB

  1. Why do we need to pull our MongoDB database string out of our server and put it into our .env?
    2. For security reasons, storing sensitive information like database credentials in an .env will protect it from being exposed in case the server code gets seen.

  2. What is middleware?
    3. Code that runs when the server gets a request but before it gets passed to your routes

  3. What does app.use(express.json()) do?
    4. Adds middleware that parses incoming requests with JSON payloads.

  4. What does the /:id mean in a route?
    5. It's a route parameter.

  5. What is the difference between PUT and PATCH?
    6. PUT is used to completely replace an existing resource with the provided data while PATCH is used to partially update an existing resource.

  6. How do you make a default value in a schema?
    7. Use the "default" property in it

  7. What does a 500 error status code mean?
    8. There was an unexpected error on the server-side while attempting to connect

  8. What is the difference between a status 200 and a status 201?
    9. Both mean that the request was successful but 200 means that the server has returned the requested resource while 201 means that a new resource has been created.

Why it matters

Because status codes provide important info about the request outcome from whatever website you're on, allowing you to troubleshoot and debug easily.
