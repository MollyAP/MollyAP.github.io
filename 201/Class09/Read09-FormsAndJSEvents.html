Forms and JS Events

HTML Forms

1.) Why are forms so important in web development?
Because they allow users to input data and submit it to the server for processing.

2.) When designing a form, what are some key things to keep in mind when it comes to user experience?
They should be easy to fill out, have clear labels, and provide helpful error messages.

3.) List 5 form elements and explain their importance.
input: used to collect user input, such as text, numbers, and checkboxes.
label: used to provide a description or label for a form input element.
select: used to create drop-down menus for users to select options from.
textarea: used for collecting longer text inputs from users.
button: used to submit or reset a form.

Learn JS

1.) How would you describe events to a non-technical friend?
Events are actions that happen in the browser, like someone clicking a button or a page finishing loading.

2.) When using the addEventListener() method, what 2 arguments will you need to provide?
The event you want to listen for like a click and the function to be executed when the event happens.

3.) Describe the event object. Why is the target within the event object useful?
It's an object that contains information about the event that happened like the target element and any user input data. It's useful for determining which element triggered the event.

4.) What is the difference between event bubbling and event capturing?
Event bubbling is when the event is first captured by the innermost element and then propagated to outer elements, event capturing is the opposite.
