Object Prototypes Using A Constructor

Explain prototypes and inheritance via an analogy from your previous work experience. NOTE: This is a very common front end developer interview question

I work at Lowes and you had a customer who wanted to buy a riding mower, I showed them a John Deere S180 which had all the features you'd expect of a lawn mower such as... cutting grass, but they want something "more advanced" with extra features like being able to handle snow but we don't have such a mower in stock (no one does) but after some research I inform the customer that It's possible to make that modification himself as we sell a snowplow attachment for that mower on our website. the basic model to add the extra features they need, In this JavaScript analogy objects are like mowers and prototypes are like the basic model which has all the essential features that all other mowers will have aka the template for creating new mowers (objects) and when the customer and I modify the basic model to add extra features (adding the snowplow attachment), we are creating a new object that inherits the properties and methods of the prototype and we can add new properties and methods to it without modifying the original.
