Why it matters:

Interfaces matter because C# lacks support for multiple class inheritance, meaning you can only inherit from a single class. Interfaces on the other hand let us define a contract / set of behaviors that a class or struct must implement and include behavior from multiple sources in a single class without the need for multiple inheritance making interfaces ideal for code reusability, modularity, and maintaining a flexible and scalable codebase.
