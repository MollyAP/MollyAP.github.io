Properties

  1. What's the difference between a static and an instance constructor?

    Static constructors are called only once per type, regardless of the number of instances created and an instance constructor is called each time a new instance of the class is created. Also static constructors are used to initialize static members of a class while instance constructors initialize instance members.

  2. How does the use of a static constructor differ from setting properties/values?

    A static constructor initializes static members of a class, while setting properties/values is usually done by instance constructors or by directly assigning values to properties or fields after creating an instance of the class.

Garbage Collection Fundamentals

  1. Compare "Garbage Collection" in C# with the lifecycle of normal household items.

    When a household item reaches the end of its usefulness or becomes obsolete, we dispose of it to free up space and keep our place clean. In C#, objects that are no longer needed or have completed their tasks are disposed of to release the memory resources they occupy.
