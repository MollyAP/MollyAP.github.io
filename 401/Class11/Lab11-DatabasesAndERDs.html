Databases and ERDs

This is a visual representation of my database schema, depicting the entities, attributes, and relationships between different tables.
  1. Primary Keys (white):

    • Amenity: ID (Primary Key)
    • Hotel: ID (Primary Key)
    • HotelRoom: ID (Primary Key)
    • Room: ID (Primary Key)
    • RoomAmenity: ID (Primary Key)

  2. Foreign Keys (gray):

    • HotelRoom: RoomID (Foreign Key referencing Room.ID)
    • HotelRoom: HotelID (Foreign Key referencing Hotel.ID)
    • RoomAmenity: RoomID (Foreign Key referencing Room.ID)
    • RoomAmenity: AmenityID (Foreign Key referencing Amenity.ID)

  3. Composite Keys:

    • There are none! :)

  4. Navigation Properties:

    1. HotelRoom

      • Room (Navigation property to Room entity, representing a Room associated with a HotelRoom)
      • Hotel (Navigation property to Hotel entity, representing a Hotel associated with a HotelRoom)

    2. RoomAmenity

      • Room (Navigation property to Room entity, representing a Room associated with a RoomAmenity)
      • Amenity (Navigation property to Amenity entity, representing an Amenity associated with a RoomAmenity)

  5. Relationships between tables:

    • Hotel and HotelRoom: One Hotel can have multiple HotelRooms, but each HotelRoom is associated with only one Hotel. (1:Many)

    • Room and HotelRoom: One Room can be associated with multiple HotelRooms, but each HotelRoom is linked to only one Room. (1:Many)

    • Room and RoomAmenity: One Room can have multiple RoomAmenities, and each RoomAmenity is linked to only one Room. (1:Many)

    • Amenity and RoomAmenity: One Amenity can be associated with multiple RoomAmenities, and each RoomAmenity is connected to only one Amenity. (1:Many)
