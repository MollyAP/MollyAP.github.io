  2. What are the different types of Database Keys?

    • What is a Primary Key?
      • A unique identifier for each record in a table that ensures each row is uniquely identifiable and serves as a reference for relationships with other tables.

    • What is a Foreign Key?
      • A field in one table that refers to the primary key of another table and establishes a link between the two tables and enforces referential integrity.

    • What is a Composite Key?
      • A combination of two or more columns that together form a unique identifier for a record in a table.

    • How are they different? When do you use 1 over the others?

      1. Primary Key: The primary key uniquely identifies each record in a table and ensures data integrity. It cannot contain duplicate or null values. Every table should have a primary key.

      2. Foreign Key: The foreign key is used to establish relationships between tables. It references the primary key of another table, creating a link between the two tables. It helps maintain referential integrity and ensures that the data between related tables remains consistent.

      3. Composite Key: A composite key is used when a single column cannot uniquely identify a record, but a combination of columns can. It is used to create a unique identifier for a table by using multiple fields together.

        4. Use a primary key when you need a unique identifier for each record in a table. Use a foreign key when establishing relationships between tables to ensure data consistency and integrity. Use a composite key when a single column cannot guarantee uniqueness, but a combination of columns can create a unique identifier.


DBMS

  1. What are Relationships in a relational database?

    • What is a 1:1 relationship?
      • One record in a table is associated with only one corresponding record in another table, and vice versa.

    • What is a Many:Many relationship?
      • Multiple records in one table are associated with multiple records in another table.

    • How about a 1: Many or a Many:1?
      • A single record in one table is associated with multiple records in another table, but each record in the second table is associated with only one record in the first table, or vice versa.
